練習專案四：找出章魚里 4

By Steve 2025-09-09

成品介紹

這是一個透過 餘弦相似度 與選舉投票數據，找出全台最像「天玉里」的專案。
利用 pandas、sqlite3 進行資料處理，並用 gradio 做出互動式成品，最後將成品部署在 Hugging Face Demo 上展示。

  • 成果：Hugging Face Demo

專案紀錄與進程

以下為本專案從資料清理到視覺化的完整實作記錄，包含每階段的筆記與驗證過程：

  • 資料分析的七個練習專案：累積個人作品集：參考資料
  • 數據清理過程：資料檢視與基礎清理
  • 概念驗證：視覺化圖表驗證與可行性測試
  • GitHub 原始碼：包含資料處理與圖表繪製程式碼

成果展示

  • 找出的最相似里：南投縣 草屯鎮 中山里
  • 相似程度：0.999998
  • 天玉里實際排名：2482 僅屬中上程度。

分析流程

  1. 資料蒐集
    • CEC「總統候選人得票數一覽表」22 份試算表
  2. 資料清理與建庫
    • 轉換為 SQLite 資料庫：taiwan_presidential_election_2024.db
  3. 相似度計算
    • 餘弦相似度公式：
      cosine similarity = (A · B) / (||A||·||B||)
  4. 互動式展示
    • Gradio App 部署於 Hugging Face Space

靜態圖表補充

  • 前 15 名相似里 vs 天玉里排名（2482） 可以看到天玉里實際排名第 2482，前 15 名的相似度皆在 0.99996 以上，差距極小，但天玉里落後許多，驗證了『名不符實』的結果。

個人反思、心得

在這次專案中，第一次接觸到 餘弦相似度，學會如何用向量比較資料的相似性。同時學習 資料庫正規化 的概念與實作方法，並體會到 以數據為導向的思考邏輯，能用資料而非直覺來驗證想法。此外，頭一次透過 Hugging Face Spaces 與 Gradio，學會新的展示與部屬方式，整體是一個有趣的練習。這次練習讓我體會到相似度分析不只可以應用在選舉數據，未來也能延伸到使用者行為分析、商品推薦等場景。

  • 學到的重點：
    • 餘弦相似度的實務應用
    • 資料庫正規化的重要性
    • 如何將分析結果做成互動式

專案復原

語言與模組

  • dependencies：
    • python 3.12.11
    • pandas 2.3.1
    • numpy 1.26.4
    • gradio 5.43.1

匯出 environment.yml

目的重現專案環境。

conda env export -n taiwan_presidential_election_2024 > environment.yml

如何重現

  • 安裝 Miniconda
  • 依據 environment.yml 建立環境： 
    conda env create -f environment.yml
  • 將 data/ 資料夾中的「總統-A05-4-候選人得票數一覽表-各投開票所」22 個試算表檔案放到專案資料夾的 data/ 資料夾中
  • 啟動環境並執行 python create_taiwan_presidential_election_2024_db.py 就能在 data/ 資料夾中建立 taiwan_presidential_election_2024.db
  • 啟動環境並執行 python app.py 並前往 http://127.0.0.1:7860 瀏覽成品。
